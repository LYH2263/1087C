
# 二、0-1 代码生成题（15 道）

**Z1. 限时秒杀活动模块**
请基于以下 PRD 从零构建限时秒杀功能。管理端可创建秒杀场次：选择参与书籍、秒杀价、秒杀库存、开始与结束时间。用户端首页展示"正在进行/即将开始"的秒杀专区，卡片显示秒杀价、原价划线、剩余库存进度条和精确到秒的倒计时；未开始显示"即将开始"、进行中可"立即抢购"、结束显示"已结束"。抢购下单走独立的秒杀库存扣减（不复用普通库存），需保证不超卖、每人限购、活动时间窗口校验。需处理倒计时到点自动切换状态、客户端与服务端时间偏差、并发抢购的原子扣减。

页面上验证：创建一个进行中的场次后，前端出现倒计时与秒杀价，抢购成功扣减秒杀库存且售罄后按钮禁用。

**Z2. 图书批量 CSV 导入**
请从零实现管理端图书批量导入。管理员上传 CSV（含书名、作者、ISBN、价格、库存、分类名、描述、封面 URL 等列），系统解析并逐行校验：必填项、价格/库存格式、ISBN 唯一性、分类是否存在（不存在可选择自动创建）。导入完成后返回结果报告：成功条数、失败条数与每条失败的行号及原因，前端以表格展示报告并支持下载错误明细。需支持事务或逐行容错（部分成功）、大文件处理、编码与表头校验、重复 ISBN 去重。

页面上验证：上传一个含部分错误行的 CSV，正确的书籍入库并出现在书籍列表，错误行在报告中给出可读原因。

**Z3. 电子发票申请与开具**
请从零构建发票模块。用户对已支付订单申请电子发票，填写抬头类型（个人/企业）、抬头名称、企业税号（企业必填且需格式校验）、邮箱。新增"我的发票"页面展示申请状态（待开具/已开具/已驳回）。管理端有发票管理列表，可开具（生成发票号与可下载的发票内容/PDF 占位）或驳回并填写原因。需处理一单仅能有一张有效发票、退款订单不可开票、重复申请拦截。后端需有发票数据模型与状态机。

页面上验证：用户提交申请后管理端可见，开具后用户侧状态变为已开具并可下载，企业抬头缺税号时被前后端拦截。

**Z4. 书单/专题合集**
请从零实现书单专题功能。管理端可创建书单（标题、封面、简介、排序），向书单中添加/移除书籍并调整书籍在书单内的顺序。用户端新增"专题"入口，展示所有上线书单卡片，点击进入书单详情页按设定顺序展示书籍并可直接加购。书单支持上线/下线状态控制。需处理书单内书籍被下架时的展示、空书单、书籍顺序的持久化。后端提供书单 CRUD 与书单-书籍关联管理接口。

页面上验证：创建并上线一个含数本书的书单后，用户端专题页出现该书单，进入可按设定顺序浏览并加购，下线后用户端不可见。

**Z5. 书籍 Q&A 问答模块**
请从零构建书籍问答功能。书籍详情页新增"问答"区，登录用户可对该书提问，管理员或已购该书的用户可回答，提问与回答按时间展示，问题可显示回答数。支持对回答点赞、按热度（点赞数）或时间排序、分页加载。管理端可删除违规问答。需处理未登录不可提问、空内容校验、XSS 转义、点赞幂等（同人不可重复点赞）。后端需设计问题、回答、点赞的数据模型与接口。

页面上验证：在某本书下提问后问答区出现该问题，回答并点赞后回答数与点赞数正确更新，按热度排序生效。

**Z6. 书籍对比模块**
请从零实现书籍对比功能。用户在书籍列表可勾选"加入对比"（最多 4 本），页面出现对比浮动栏显示已选数量；进入对比页以并排表格对比书名、作者、价格、库存、分类、销量、评分、ISBN 等字段，差异项高亮，可移除单项或清空，并支持从对比页直接加购。对比选择需在本地持久化（刷新不丢失）。需处理超出上限的提示、重复加入、已下架书籍在对比中的处理。

页面上验证：勾选 2~3 本书后对比栏计数变化，进入对比页并排展示且差异高亮，移除后表格同步更新。

**Z7. 标签系统与标签云筛选**
请从零构建图书标签能力。管理端可创建标签并为书籍打多个标签（如"畅销""新书""获奖"）。书籍列表页新增标签云区域，标签按关联书籍数量呈现不同字号；点击标签筛选出含该标签的书籍，支持多标签组合筛选（与/或逻辑）并与现有的书名/作者/价格筛选共存。书籍卡片展示其标签。需设计书籍-标签多对多模型与查询，处理标签删除时的关联清理、组合筛选的查询构造。

页面上验证：给若干书籍打标签后标签云出现并按数量分级，点击标签正确过滤列表，多标签组合筛选结果自洽。

**Z8. 新书预售与到货通知**
请从零实现预售功能。管理端可将书籍设为预售并配置预计到货日期与预售库存，用户端预售书籍卡片显示"预售"标签与到货日期、可"预约/预订"。到货日期前不计入普通库存销售；管理员标记"已到货"后转为正常在售并向所有预约用户生成到货通知。新增"我的预约"页面。需处理预售期下单的特殊状态、到货后状态切换、重复预约幂等、预约用户的批量通知。

页面上验证：设置一本预售书后用户端出现预售标签并可预约，管理员标记到货后该书转为可正常购买且预约用户收到到货通知。

**Z9. 满额包邮与凑单提示**
请从零构建运费规则与凑单引擎。后端可配置运费规则（如订单满 X 元包邮，否则收取固定/按件运费）。购物车与结算页实时计算运费并展示，未达包邮门槛时提示"再买 X 元即可包邮"并推荐若干凑单书籍（价格接近差额）；下单时将运费计入订单总额并在订单详情拆分展示商品金额与运费。需处理规则可配置、勾选结算时按所选金额计算运费、推荐书籍排除购物车已有项与下架项。

页面上验证：购物车未达门槛时显示差额提示与凑单推荐，加购凑单书后达到门槛运费归零，订单详情正确展示运费明细。

**Z10. 评价图片上传与画廊点赞**
请从零扩展评价为图文评价。用户对已完成订单评价时可上传至多若干张图片（复用/参考现有 multer 上传能力），书籍详情评价列表展示带缩略图的图文评价，点击图片进入画廊大图预览（左右切换）。其他用户可对评价点赞，评价支持"有图优先/最新/最赞"排序。需处理图片大小/类型校验、上传失败回滚、点赞幂等、画廊键盘与触摸切换、XSS。后端需扩展评价模型存储图片与点赞。

页面上验证：提交带图评价后详情页出现缩略图，点击可大图浏览，点赞数实时更新且有图评价可优先排序。

**Z11. 后台批量上下架与批量改价**
请从零实现管理端批量操作。书籍管理列表每行增加复选框，支持全选/跨筛选条件选择当前结果集；底部批量操作栏提供"批量上架/下架""批量改价"（按固定值或按百分比涨跌价，含价格下限保护）"批量设置分类"。操作前弹出确认并展示影响条数，执行在事务中完成并返回成功/失败明细，价格变更需记录。需处理部分失败的回滚策略、空选禁用、批量改价的浮点精度（分为单位）。


页面上验证：勾选多本书批量下架后列表状态同步变化，批量按百分比改价后价格按规则更新且不低于下限。

**Z12. 收货地址智能解析**
请从零构建地址智能录入。地址新增/编辑表单提供一个"智能填写"输入框，用户粘贴一段完整文本（如"张三 13800001111 浙江省杭州市西湖区文三路 100 号"），点击解析后自动拆分并填充收件人、手机号、省、市、详细地址等字段，识别失败的字段给出提示由用户手动补全。解析需处理手机号正则提取、省市识别、姓名与地址分离、多种文本顺序与含/不含分隔符的情况。后端或前端实现解析逻辑均可，但需有可测试的纯函数。


页面上验证：粘贴多种格式的地址文本点击解析后，各字段被正确拆分填充，异常文本给出明确的未识别提示。

**Z13. 储值余额与余额支付**
请从零实现账户余额体系。用户有账户余额，可在"我的钱包"页面查看余额与流水，并通过"模拟充值"增加余额。结算时支付方式新增"余额支付"，下单/支付时校验余额是否充足并原子扣减、生成消费流水；订单取消或退款时退回余额并记录流水。需保证余额扣减/退回的并发安全与幂等（避免重复扣或重复退），流水需可追溯每笔变动来源。后端需有钱包与流水模型。


页面上验证：充值后余额增加，用余额支付一笔订单后余额减少且流水新增，退款后余额回退并有对应流水记录。

**Z14. 个人资料中心与头像上传**
请从零构建用户资料中心。新增"个人中心"页面展示并允许修改昵称、邮箱、手机号（邮箱/手机改动需唯一性校验），上传/裁剪头像（复用上传能力），以及修改密码（校验原密码、新密码强度、二次确认）。顶部导航的用户区显示头像与昵称。需处理唯一性冲突的友好提示、头像上传的类型/大小校验、修改密码后会话处理。后端需扩展用户资料更新与改密接口（注意密码哈希）。


页面上验证：修改昵称/头像后导航区即时更新，改密后用新密码可登录、旧密码失效，邮箱与已存在账号冲突时被拦截。

**Z15. 销售目标设定与完成度**
请从零实现运营目标管理。管理端可按月设定销售额目标和订单量目标，新增"目标管理"页面展示当月目标、实际完成值、完成百分比进度条与达成预测（按当前进度线性外推月末值），并展示历史各月目标的达成情况列表。实际完成值需基于有效订单（PAID/SHIPPED/COMPLETED）实时聚合。需处理目标的新增/编辑、跨月切换、无目标月份的空态、退款对完成值的影响。后端需有目标模型与聚合接口。

页面上验证：设定本月目标后进度条按真实成交额填充并显示百分比与预测，产生新订单后完成度上升。
